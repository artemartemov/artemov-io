### The Project Brief

**Custom feeds**: are an improvement from legacy feed pages, which presented content in a reverse-chron list with limited options for styling or promoting specific high-value articles/galleries.

<Box sx={{variant: 'layouts.textPage'}}>

### The Project Brief

**Custom feed creation tool**: allows editors to curate blocks of content, that they can use to build customized homepages, and other landing pages on the brands respective website.

</Box>

---

<Box sx={{variant: 'layouts.textPage'}}>

### The Project Brief

**Custom feeds** will offer a **modular** approach for managing content on landing feed pages to support more **visually compelling layouts** and present content in a **curated** and unique way.

</Box>

---

<Box sx={{variant: 'layouts.textPage'}}>

### The Project Brief

The **complexity** of styling a new site, or redesigning current sites **should not** be impacted by this new feature.

# The Blocks: Conclusion 🏁

</Box>

---

<Box sx={{variant: 'layouts.textPage'}}>

#### The Blocks: Conclusion

Having a consice set of blocks allowed the editors to quickly create content for their brands. One of the main pain points that we had to consider while creating these blocks was for editors to not spend too much time on these pages.

</Box>

---

<Box sx={{variant: 'layouts.textPage'}}>

#### The Blocks: Conclusion

By only have 4 types of blocks, the editors quickly were able to design unique landing pages. Each block type has many different layout options so the flow of content does not look or feel repetative.

</Box>

---

<Box sx={{variant: 'layouts.textPage'}}>

#### The Blocks: Conclusion

A **HUGE** win 🥳 for us turned out to be that each of the 4 block types shared the same atoms (image, headline, dek, byline and date). This allowed for spinning up a new brand with unique styles quick and efficient.

</Box>

## Layouts -> Blocks -> Items

</Box>

---

<Box sx={{variant: 'layouts.splitPage'}}>

#### Layouts

- **Layouts** are styles are what controls the structure of blocks. These are globally set styles that have no theming.

</Box>

---

<Box sx={{variant: 'layouts.splitPage'}}>

#### Layouts -> Blocks

- **Blocks** contain specific styles for the block types. This can be background styles, borders, and how much they break out of our grid.

</Box>

---

<Box sx={{variant: 'layouts.splitPage'}}>

#### Layouts -> Blocks -> Items

- **Items** are styles specific to each individual article or story. These house the styles for Headlines, bylines, deks, and dates

</Box>

```scss
// Global layout styles
.feed-block-with-ad {
	width: 100vw;

	@include respond-to( $breakpoint-tablet ) {
		.feed-block-header {
			grid-column: 1 / 3;
		}
	}

	.feed-block-content {
		margin-left: -$mobile-outer-gutter;
	}
}

// Shared global styles for custom block header container
.feed-block-header {
	margin-bottom: $vertical-rhythm-outer;
	@extend %custom-feed-block-header !optional;

	&.icon-header {
		@extend %custom-feed-block-header-icon-container !optional;

		& .block-icon {
			@extend %custom-feed-block-header-icon !optional;
		}
	}
}

// Brand styles for custom block header container
%custom-feed-block-header {
	position: relative;
	margin-bottom: $margin-s;
	padding-top: $margin-s;
	overflow: hidden;

	@include respond-to( $breakpoint-tablet-wide ) {
		margin-bottom: $vertical-rhythm-outer;
		padding-top: $vertical-rhythm-inner;
	}
}
```
