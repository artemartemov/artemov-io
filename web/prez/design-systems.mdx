# The Problem

As we are developing a new content publishing platform called **Scribe**, we are shifting focus from an **internal tool - to a SaaS product**.

It quickly became apparent that our current set of design tools **can no longer empower us** to build out a scalable product.

</Box>

---

<Box sx={{variant: 'layouts.textPage'}}>

### The Problem cont.

We also do not know the type of applications or "services" that will need to be powered _by_ scribe. We knew that whatever system we build should be device and platform agnostic.

This is why we went with a tool stack of: Figma, styled-systems, css-variables, clear component naming structure where the code is the source of truth, and working on implementing tooling that connects code and design in a bi-directional manner.

</Box>
